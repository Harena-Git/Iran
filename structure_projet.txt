STRUCTURE DU PROJET - MINI PROJET WEB DESIGN (PHP + POSTGRES)

1.  RACINE DU PROJET

/iran-news/ │ ├── docker/ ├── public/ ├── app/ ├── config/ ├──
templates/ ├── uploads/ ├── logs/ ├── .env ├── docker-compose.yml └──
README.md

2.  DOSSIER DOCKER

/docker/ ├── php/ │ └── Dockerfile ├── nginx/ │ └── default.conf └──
postgres/ └── init.sql

3.  DOSSIER PUBLIC (FRONT CONTROLLER)

/public/ ├── index.php ├── .htaccess ├── assets/ │ ├── css/ │ ├── js/ │
└── images/

4.  DOSSIER APP (LOGIQUE METIER)

/app/ ├── controllers/ │ ├── HomeController.php │ ├──
ArticleController.php │ └── AdminController.php │ ├── models/ │ ├──
Article.php │ ├── Category.php │ ├── User.php │ └── Tag.php │ ├──
services/ │ └── Database.php

5.  DOSSIER CONFIG

/config/ └── database.php

6.  DOSSIER TEMPLATES (VUES)

/templates/ ├── layout/ │ ├── header.php │ └── footer.php │ ├── front/ │
├── home.php │ ├── article.php │ └── category.php │ └── admin/ ├──
login.php ├── dashboard.php ├── articles.php └── edit_article.php

7.  DOSSIER UPLOADS

/uploads/ └── images/

8.  ROUTAGE (URL REWRITING)

.htaccess : - / => index.php - /article/slug => ArticleController -
/category/slug => CategoryController - /admin => AdminController

9.  ORGANISATION GENERALE

-   public = accessible navigateur
-   app = logique backend
-   templates = affichage
-   config = configuration
-   docker = environnement
-   uploads = fichiers utilisateurs

10. CONSEILS

-   Toujours utiliser des slugs pour SEO
-   Séparer logique et affichage
-   Ne jamais mettre de logique dans templates
-   Utiliser PDO pour PostgreSQL
-   Protéger les accès admin avec session
